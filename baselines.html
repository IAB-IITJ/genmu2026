Legacy baselines and new-track release notes

Task 3 retains the four published concept-unlearning baselines. For the face and speech tracks, the listed checkpoints are optional starting points: participants may use any suitable model and must document the exact model, checkpoint, and inference interface.

Task 3 baselines — Concept Unlearning

The table below lists the four published baseline methods retained from the original Genμ challenge, with full citations.

MethodFull nameSettingCitationStatus
ESD Erased Stable Diffusion Model-weight concept erasure using negative guidance as the training signal Erasing Concepts from Diffusion Models, Gandikota et al., 2023 Retained
CA Concept Ablation Maps the target concept distribution to an anchor concept while preserving closely related concepts Ablating Concepts in Text-to-Image Diffusion Models, Kumari et al., 2023 Retained
FMN Forget-Me-Not Efficient targeted removal of identities, objects, or styles while retaining unrelated generation capability Forget-Me-Not: Learning to Forget in Text-to-Image Diffusion Models, Zhang et al., 2023 Retained
FADE Fine-Grained Attenuation for Diffusion Erasure Adjacency-aware concept unlearning using a concept neighborhood and mesh modules Fine-Grained Erasure in Text-to-Image Diffusion-based Foundation Models, Thakral et al., 2025 · Code Retained

Task 1 setup — Face Unlearning Track

Participants may choose any face-conditioned generative model. The evaluation protocol is model-independent; submissions must expose a reproducible generation procedure and identify the exact upstream checkpoint.

Data

CelebA and FFHQ

The public validation identities are listed on the evaluation page. Teams may use the released data under its original terms and must document any additional training data.

Optional starting points

Suggested Hugging Face checkpoints

  • FoivosPar/Arc2Face
  • InstantX/InstantID
  • h94/IP-Adapter-FaceID

These recommendations do not restrict model choice.

Task 2 setup — Speech Unlearning Track

Participants may choose any TTS or voice-cloning model that can be evaluated under the released speaker protocol. The repository must document sample rate, speaker-conditioning interface, and exact checkpoint.

Data

LibriTTS / LibriTTS-R validation speakers

The public forget and retain speaker groups are listed on the evaluation page. The groups use same-gender speakers from the clean development split to make retention deliberately challenging.

Optional starting points

Suggested Hugging Face checkpoints

  • yl4579/StyleTTS2-LibriTTS
  • coqui/XTTS-v2

Other compatible models are welcome.
